# 🎴 SillyTavern Multitools

[Tiếng Việt](README.md) | [English](README.en.md) | **简体中文** | [繁體中文](README.zh-TW.md)

> **一套用于翻译与制作 SillyTavern 角色卡的 5 合 1 工具 —— 完全在你自己的电脑上运行。**
> 由 **Guillichan × Sky** 打造。

专为**翻译者**和角色卡作者设计：用 AI 翻译角色卡，同时**不破坏代码、正则、世界书和宏**（如 `{{char}}` `{{user}}`）。

一切都在本地运行 —— 你的 **API 密钥不会被发送到任何地方**，没有任何中间服务器。

> 🌐 **语言与反馈：** 顶部**页眉**（所有工具通用）带有**界面语言切换 —— Tiếng Việt / English / 中文**，以及**🐞 报告错误**按钮（打开共享的报错表格，供大家记录 bug）。翻译卡、改卡、创建预设以及创建卡的主要面板均已支持三语；高级 MVU-ZOD Studio 面板和提取卡目前仍为越南语。

## 📦 里面有什么？

打开应用后，左侧栏可在 5 个工具之间切换。**随意切换 —— 正在运行的任务不会中断。**

| | 工具 | 用途 |
|---|---|---|
| 🌐 | **Dịch Card**（翻译卡） | 翻译角色卡。*这是主力工具。* |
| 🃏 | **Tạo Card**（创建卡） | 从小说生成新卡，另含世界书、正则、MVU/ZOD 变量、游戏界面。 |
| 🎛️ | **Tạo Preset**（创建预设） | 通过与 AI 对话，创建 / 修改 SillyTavern 预设与正则脚本。 |
| 🛠️ | **Mod Card**（改卡） | 按你的要求修改 / 扩写现有角色卡（续写、深挖……）。 |
| 🔍 | **Trích Card**（提取卡） | 读取小说 → 自动提取角色与世界书条目。 |

## 🚀 首次安装（只需做一次）

### 第 1 步 —— 安装两个基础软件
- **[Node.js](https://nodejs.org/)** —— 请选 **20 或更高**版本（像普通软件一样安装即可）。
- **[Git](https://git-scm.com/downloads)** —— 默认选项即可。

> 装完后建议**重启一次电脑**，以防万一。

### 第 2 步 —— 下载源码
打开**命令提示符**（按 Windows 键 → 输入 `cmd` → 回车），然后逐行粘贴：

```bash
cd C:\
git clone https://github.com/kubi2811/ST-Card-Translation-Sky.git
```

### 第 3 步 —— 运行
进入刚下载的文件夹，**双击 `start.bat`**。

- 第一次运行会**自动安装依赖**（需要几分钟 —— 请耐心等待，不要关闭）。
- 完成后会自动打开浏览器：**http://localhost:5173**。
- 会弹出几个**黑色小窗口** —— 那是 3 个辅助工具。**请不要关闭它们。**

> 之后每次只要双击 **`start.bat`** 就行了。

## 📖 翻译一张卡 —— 5 步

### 1️⃣ 填入 API 密钥
左栏 **API Configuration**：
- 选择供应商：**Gemini / OpenAI / Claude / DeepSeek / Qwen**，或你自己的代理。
- 粘贴 **API Key**。可以粘贴**多个密钥**（每行一个）→ 速度快很多。
- 选择 **Model** → 点击 **Test Connection** → 出现绿色提示就说明成功。

> 💡 密钥越多、供应商越多 → 并行通道越多 → 翻译越快。应用会**自动遵守你的 RPM 限制**，不会被限流。

### 2️⃣ 载入角色卡
在 **Character Card** 区域：**拖放** `.json` 或 `.png` 文件，或**粘贴链接**后点 **Tải**（载入）。

### 3️⃣ 选择要翻译的内容
勾选分组：核心、开场白、世界书、关键词、正则、脚本…（全部保持勾选也没问题）。

### 4️⃣ 点击 **Dịch**（翻译）
- 实时查看进度。随时可以 **暂停** 或 **停止**。
- 进度会**自动保存** —— 不小心关掉标签页，重开还在。

### 5️⃣ 检查并导出
- 点击 **Kiểm Tra Lỗi Dịch**（检查翻译错误）→ 应用会自动扫描：未翻译的文本、损坏的 HTML/JSON、丢失的宏、MVU 变量不匹配等。提供 **Sửa nhanh**（快速修复 —— 即时、不消耗 API）和 **AI Sửa**（AI 修复，用于疑难问题）。
- 点击 **Xuất**（导出）→ 下载 `.json` 或 `.png`（重新嵌回原始卡图）→ 直接丢进 SillyTavern 就能用。

## ✨ 为什么翻译者会喜欢它

### 🔪 "外科手术式"翻译 —— 卡不会坏
应用**只翻译正文文字**，完整保留 HTML/CSS/JS、正则、链接、变量以及 `{{char}}` `{{user}}` 之类的宏。手动翻译或用普通 AI 翻译时，坏卡基本都坏在这里。

### ⚡ 多通道并行 —— 快好几倍
多个密钥 × 多个供应商 = 大量请求并行。某个通道一空闲就**立刻领取下一条**，不会干等着某个慢条目。同时严格遵守 **RPM 限制** —— 不会出现 429 错误。

### 🔀 卡片对比 + 智能合并 ⭐ *（重点功能）*

**场景：** 你把一张卡翻译完了，结果**作者更新了原版**。以前只能整张卡重新翻译。

现在点击 **"🔀 So Sánh Card"**（卡片对比 —— 就在 *Character Card* 上方的按钮），载入 3 个文件：

| 槽位 | 含义 |
|---|---|
| **Card Raw** | **旧版**原文卡 |
| **Card Đã Dịch** | 你**之前的译文**卡 |
| **Card Final** | **新版**原文卡（作者刚更新的） |

点击 **"Gộp thông minh"**（智能合并），应用会逐条对比：
- 条目**没变** → **复用你的旧译文**（正则/代码原样保留 → **不会出错**）。
- 条目**新增或被修改** → 保留原文，等待翻译。

界面会实时预览（♻ 绿色 = 复用，✏️ 橙色 = 待翻译），并显示计数：**"复用 100 · 待翻译 10"**。然后点 **"Đưa sang Dịch Card"**（送去翻译卡）→ 只翻译那 10 条新内容。**极大节省时间。**

> 对比界面还支持逐条并排查看 3 个版本、**就地编辑并保存**，以及**"仅显示差异"**筛选。

### 📖 轻量翻译 vs 完整翻译
**翻译配置**里有两个预设：**「📖 完整翻译」**和**「⚡ 轻量翻译」**——后者只翻译**关键词 + 正则 + 开场白 + 卡名 / 世界书条目名 + MVU 变量名**，保留大段正文为原文（AI 仍能读懂并用目标语言回复）。对于重度 MVU 卡，轻量翻译**快好几倍**。

### 🧠 保持一致性
- **术语表（Glossary）** —— 强制 AI 按你的要求翻译人名和专有名词。
- **翻译记忆** —— 相同的句子得到相同的译文。
- **MVU / EJS 同步** —— 代码里和世界书里的变量名始终保持一致。

## ❓ 常见问题

**点"更新"报错 / 卡住**
→ 见上文[更新版本](#-更新版本)，用那 3 条手动命令。

**应用提示 `Failed to resolve import ...`**
→ 新版本增加了依赖库。彻底关闭应用后重新运行 **`start.bat`**（它会自动安装）。仍不行就在安装目录里运行 `npm install`。

**载入卡之后应用卡顿几秒**
→ 那张卡的正则脚本非常大（几百 KB）。这是正常现象，稍等即可。若不需要翻译脚本，可在第 3 步取消勾选 **Regex** 分组。

**连不上 API / 报 CORS 错误**
→ 检查 Base URL 与 Key，试试开启 **CORS Proxy**，或点 **Test Connection** 查看具体错误。

**扩写超大条目时 Gemini 返回 524 / 超时**
→ 单次请求耗时过长，代理超时了。请对整个段落使用**扩写模式**（应用会自动分段），或改选一个更小的区块来深挖。

**`start.bat` 弹出的黑色小窗口是什么？**
→ 是 3 个辅助工具（创建卡 / 创建预设 / 改卡）。**请勿关闭**，否则那几个工具打不开。
