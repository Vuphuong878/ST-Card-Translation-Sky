# 🎴 SillyTavern Multitools

[Tiếng Việt](README.md) | [English](README.en.md) | [简体中文](README.zh-CN.md) | **繁體中文**

> **一套用於翻譯與製作 SillyTavern 角色卡的 5 合 1 工具 —— 完全在你自己的電腦上執行。**
> 由 **Guillichan × Sky** 打造。

專為**翻譯者**與角色卡作者設計：用 AI 翻譯角色卡，同時**不破壞程式碼、正規表示式、世界書與巨集**（如 `{{char}}` `{{user}}`）。

一切都在本機執行 —— 你的 **API 金鑰不會被傳送到任何地方**，沒有任何中間伺服器。

> 🌐 **語言與回報：** 頂部**頁首**（所有工具通用）具有**介面語言切換 —— Tiếng Việt / English / 中文**，以及**🐞 回報錯誤**按鈕（開啟共用的錯誤回報表格，供大家記錄 bug）。翻譯卡、改卡、建立預設以及建立卡的主要面板皆已支援三語；進階 MVU-ZOD Studio 面板與擷取卡目前仍為越南語。

## 🚀 首次安裝（只需做一次）

### 第 1 步 —— 安裝兩個基礎軟體
- **[Node.js](https://nodejs.org/)** —— 請選 **20 或更新**的版本（像一般軟體那樣安裝即可）。
- **[Git](https://git-scm.com/downloads)** —— 使用預設選項即可。

> 裝完後建議**重新開機一次**，以防萬一。

### 第 2 步 —— 下載原始碼
開啟**命令提示字元**（按 Windows 鍵 → 輸入 `cmd` → Enter），然後逐行貼上：

```bash
cd C:\
git clone https://github.com/kubi2811/ST-Card-Translation-Sky.git
```

### 第 3 步 —— 執行
進入剛下載的資料夾，**按兩下 `start.bat`**。

- 第一次執行會**自動安裝相依套件**（需要幾分鐘 —— 請耐心等待，不要關閉）。
- 完成後會自動開啟瀏覽器：**http://localhost:5173**。
- 會跳出幾個**黑色小視窗** —— 那是 3 個輔助工具。**請不要關閉它們。**

> 之後每次只要按兩下 **`start.bat`** 就好了。

## 📖 翻譯一張卡 —— 5 步

### 1️⃣ 填入 API 金鑰
左欄 **API Configuration**：
- 選擇供應商：**Gemini / OpenAI / Claude / DeepSeek / Qwen**，或你自己的 Proxy。
- 貼上 **API Key**。可以貼上**多組金鑰**（每行一組）→ 速度快很多。
- 選擇 **Model** → 按 **Test Connection** → 出現綠色提示就表示成功。

> 💡 金鑰越多、供應商越多 → 並行通道越多 → 翻譯越快。應用程式會**自動遵守你的 RPM 限制**，不會被限流。

### 2️⃣ 載入角色卡
在 **Character Card** 區域：**拖放** `.json` 或 `.png` 檔案，或**貼上連結**後按 **Tải**（載入）。

### 3️⃣ 選擇要翻譯的內容
勾選群組：核心、開場白、世界書、關鍵字、正規表示式、腳本…（全部保持勾選也沒問題）。

### 4️⃣ 按下 **Dịch**（翻譯）
- 即時查看進度。隨時可以 **暫停** 或 **停止**。
- 進度會**自動儲存** —— 不小心關掉分頁，重開還在。

### 5️⃣ 檢查並匯出
- 按 **Kiểm Tra Lỗi Dịch**（檢查翻譯錯誤）→ 應用程式會自動掃描：未翻譯的文字、損壞的 HTML/JSON、遺失的巨集、MVU 變數不一致等。提供 **Sửa nhanh**（快速修復 —— 即時、不消耗 API）與 **AI Sửa**（AI 修復，用於疑難問題）。
- 按 **Xuất**（匯出）→ 下載 `.json` 或 `.png`（重新嵌回原始卡圖）→ 直接丟進 SillyTavern 就能用。

## ✨ 為什麼翻譯者會喜歡它

### 🔪 「外科手術式」翻譯 —— 卡不會壞
應用程式**只翻譯正文文字**，完整保留 HTML/CSS/JS、正規表示式、連結、變數以及 `{{char}}` `{{user}}` 之類的巨集。手動翻譯或用一般 AI 翻譯時，壞卡幾乎都壞在這裡。

### ⚡ 多通道並行 —— 快上好幾倍
多組金鑰 × 多個供應商 = 大量請求並行。某個通道一空閒就**立刻領取下一筆**，不會乾等某個慢條目。同時嚴格遵守 **RPM 限制** —— 不會出現 429 錯誤。

### 🔀 卡片比對 + 智慧合併 ⭐ *（重點功能）*

**情境：** 你把一張卡翻譯完了，結果**作者更新了原版**。以前只能整張卡重新翻譯。

現在按下 **「🔀 So Sánh Card」**（卡片比對 —— 就在 *Character Card* 上方的按鈕），載入 3 個檔案：

| 欄位 | 意義 |
|---|---|
| **Card Raw** | **舊版**原文卡 |
| **Card Đã Dịch** | 你**先前的譯文**卡 |
| **Card Final** | **新版**原文卡（作者剛更新的） |

按下 **「Gộp thông minh」**（智慧合併），應用程式會逐條比對：
- 條目**沒變** → **沿用你的舊譯文**（正規表示式／程式碼原樣保留 → **不會出錯**）。
- 條目**新增或被修改** → 保留原文，等待翻譯。

介面會即時預覽（♻ 綠色 = 沿用，✏️ 橘色 = 待翻譯），並顯示計數：**「沿用 100 · 待翻譯 10」**。接著按 **「Đưa sang Dịch Card」**（送去翻譯卡）→ 只翻譯那 10 筆新內容。**大幅節省時間。**

> 比對畫面還支援逐條並排檢視 3 個版本、**就地編輯並儲存**，以及**「只顯示差異」**篩選。

### 📖 輕量翻譯 vs 完整翻譯
**翻譯設定**裡有兩個預設：**「📖 完整翻譯」**和**「⚡ 輕量翻譯」**——後者只翻譯**關鍵詞 + 正則 + 開場白 + 卡名 / 世界書條目名 + MVU 變數名**，保留大段正文為原文（AI 仍能讀懂並用目標語言回覆）。對於重度 MVU 卡，輕量翻譯**快好幾倍**。

### 🧠 維持一致性
- **術語表（Glossary）** —— 強制 AI 依你的要求翻譯人名與專有名詞。
- **翻譯記憶** —— 相同的句子得到相同的譯文。
- **MVU / EJS 同步** —— 程式碼裡與世界書裡的變數名稱始終保持一致。

## ❓ 常見問題

**按「更新」出錯 / 卡住**
→ 見上方[更新版本](#-更新版本)，改用那 3 條手動指令。

**應用程式顯示 `Failed to resolve import ...`**
→ 新版本增加了相依套件。完全關閉應用程式後重新執行 **`start.bat`**（它會自動安裝）。仍不行就在安裝目錄執行 `npm install`。

**載入卡之後應用程式卡頓幾秒**
→ 那張卡的正規表示式腳本非常大（好幾百 KB）。這是正常現象，稍等即可。若不需要翻譯腳本，可在第 3 步取消勾選 **Regex** 群組。

**連不上 API / 出現 CORS 錯誤**
→ 檢查 Base URL 與 Key，試著開啟 **CORS Proxy**，或按 **Test Connection** 查看確切錯誤。

**擴寫超大條目時 Gemini 回傳 524 / 逾時**
→ 單次請求耗時過長，Proxy 逾時了。請對整個段落使用**擴寫模式**（應用程式會自動分段），或改選一個較小的區塊來深掘。

**`start.bat` 跳出的黑色小視窗是什麼？**
→ 是 3 個輔助工具（建立卡 / 建立預設 / 改卡）。**請勿關閉**，否則那幾個工具打不開。
